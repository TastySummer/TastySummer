创建对象

 2018/01/18   Share

工厂模式

  • 优点：可以无限次调用这个函数创建多个相似对象
  • 缺点：没有解决对象识别问题

构造函数模式

要创建新的 Person 实例，必须使用 new 操作符。以这种方式调用构造函数会经历以下 4 个步骤：

  • 创建一个新的对象
  • 将构造函数作用域赋给新的对象（this 指向这个新的对象）
  • 执行构造函数中的代码
  • 返回新的对象

构造函数与其他函数的唯一区别就是调用它的方式不同。

  • 优点：通过创建自定义构造函数来创建对象，可以将它的实例即这些对象标识为同一种特定的类型。
  • 缺陷：每创建一个 Person 实例就会创建一个不同的 Function 实例，即不同实例上的同名函数是不相等的，会导致不同的作用域链和标识符解析。解决这样的问题可以将函数定义在构造函数外。

  • 新缺陷1：如果该构造函数中包含很多方法，则需要定义相应数量的全局函数，且只为某个对象使用；
  • 新缺陷2：全局函数的大量使用，使得该引用类型毫无封装性可言；

原型模式

我们创建的一个构造函数，就会创建一个prototype属性指向该函数的原型对象（包含该构造函数所有实例的共享方法和属性）；

当调用构造函数创建一个新的实例时，该实例内部会包含一个 [[Prototype]] 指针指向原型对象。[[Prototype]] 无法访问。虽然 [[Prototype]] 无法访问，我们可以通过调用原型对象的 isPrototypeOf() 的方法来确定实例与原型对象之间是否存在这种指向关系。

此外，ECMAScript5 中定义了一个方法 Object.getPrototypeOf() 来返回 [[Prototype]] 值。

我们显然可以通过 Object.getPrototypeOf() 方法获得对象实例的原型，却不能重写原型中的值。如果实例中添加一个属性与原型中属性同名，则实例属性屏蔽原型属性，但不会修改原型中的那个属性。使用 delete 操作符可以完全删除该实例属性，恢复指向原型连接。

检测属性是否存在于实例中或者原型中的方法

  • 使用 hasOwnProperty() 方法检测属性是否存在实例中。hasOwnProperty() 方法是从 Object 继承而来的，是实例的方法。

  • 单独使用 in 操作符

in 操作符会通过对象访问给定属性时返回 true，无论该特定属性是存在原型中还是实例中。

因此同时使用 hasOwnProperty() 方法和 in，就可以确定某属性是存在原型中还是实例中。

for-in 循环

for-in 循环返回所有能够通过对象访问的，可枚举的属性，其中包括存在于实例或者原型中的属性。原型中被实例屏蔽的不可枚举属性也会在 for-in 循环中返回。因为实例中屏蔽原型属性的同名属性为开发人员定义属性，开发人员定义属性均可枚举。（这段话在原书中我读了好几遍。。。才摆弄明白。。。）

ECMAScript5 获取所有可枚举实例属性的方法 Object.keys()，该方法接受一个对象作为参数，返回一个字符串数组。

获得所有实例属性的方法，Object.getOwnPropertyNames()

原型模式简化

  • 缺陷：由于此种方法相当于完全重写了默认的原型对象，因此原型对象的 constructior 指向 Object 构造函数。
  • 解决办法：重新定义 constructor 属性。

  • 缺陷：重写原型对象构造函数和最初原型的联系，也切断了现有原型与任何之前已经存在的对象实例之间的联系。之前已经存在的对象实例引用的仍是最初原型。

原生对象的原型

我们不推荐在修改原生对象的原型，会出现意想不到的问题。

原型对象的优点和缺点

  • 优点：不必在构造函数中定义属性和方法，可以通过构造函数直接在原型中定义所有实例共享该原型的属性和方法。
  • 缺陷：如果原型中的属性时引用类型时，当修改某个实例的该属性引用，由于原型模式的共享性，会导致所有实例该属性引用均产生变化。

组合使用构造函数模式和原型模式（最常用，定义引用类型的默认模式）

  • 构造函数模式用于定义实例属性
  • 原型模式用于定义方法和共享属性

  • 优点：每个实例都会有自己的一份实例属性的副本，但同时又共享着对方法的引用，最大程度地节省了内存。同时混合模式还支持向构造函数传递参数。

动态原型模式（在构造函数中初始化原型）

  • 优点：封装性

寄生构造函数模式

  • 不能算作优点的优点：由于不能直接修改原生构造函数，我们可以使用寄生构造函数创建一个具有额外方法的特殊引用类型。
  • 缺陷：构造函数返回的对象与原型和构造函数都没有联系，不能用 instanceof 来确定对象类型。

稳妥构造函数模式（遵循寄生构造函数模式，不用 this 和 new）

稳妥对象(durable object)：没有公共属性的对象。适用于禁止使用 this 和 new 的环境。

  • 除调用 sayName() 外，无法访问 person 中的数据成员；

总结

创建对象最常用的方式为组合模式
